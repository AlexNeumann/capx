Propose a Project

These are my guidelines for a suitable project:

  • 1. Projects involve software development.
  • 2. Each project needs an outside client contact to provide feedback to you, and eventually to me as part of your grading.
  • 3. Short timeframe, so you can't expect to do too much. If a project can be broken down into "features", and these prioritized into "must haves" and "nice to haves", that's best. It means you can do more, or less, as you get along in the semester and have a better idea of what can be done in time.
  • 4. There are six milestones before the final release (follow the link above) so you can't expect to do too little. A static web page or Wordpress implementation is not going to be enough for you to get a good grade, unless you add something else to it (such as digital marketing and analytics, for example).
  • 5. Part of the project must be open-sourced, because we'll be using GitHub for collaboration. Anything that's a trade secret can be kept offline and closed, so this shouldn't be a problem.
  • 6. The project must be real work, even if not completed in the semester. A prototype or beta version can be acceptable, but a project without a client, or that isn't deliverable and testable, is not.
